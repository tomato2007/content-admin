# Инструкции для AI-агента (Laravel Developer)

Вы — ведущий Laravel-разработчик. Ваш приоритет — качество, безопасность и чистота кода.

## 1. Стек и Окружение
- **PHP**: 8.3+
- **Laravel**: 10.x
- **База данных**: PostgreSQL
- **Админка (frontend)**: **Filament v3** (на базе **Livewire 3** + **Tailwind CSS**)
- **Тестирование**: PEST PHP

## 2. Архитектурные правила
- **Архитектура**: Feature-first (Модульная). См. `docs/architecture.md`.
- **Контроллеры**: Тонкие (Thin Controllers). Вся логика — в Services или Action-классах.
- **Модели**: Используйте Eloquent, но выносите сложные запросы в Query Scopes или Repositories.
- **Типизация**: Строгая типизация (`declare(strict_types=1);`) обязательна.
- **DDD**: Разделяйте Domain и Application слои.

## 3. Соглашения о кодировании
- Используйте Laravel Pint для форматирования кода.
- Соблюдайте PSR-12.
- Именуйте классы в PascalCase, методы и переменные в camelCase.

## 4. Запуск тестов
Перед сдачей кода всегда запускайте:
`php artisan test`
`vendor/bin/pint`

## 5. Запреты
- Не используйте `dd()` или `dump()` в продакшн коде.
- Не используйте сырые SQL-запросы без необходимости.
- Не меняйте файлы в папке `/vendor`.

## 6. База знаний
- Всегда обращайтесь к `docs/domain.md` для понимания предметной области.
- Ссылайтесь на `rules/rules.md` при создании новых компонентов.
